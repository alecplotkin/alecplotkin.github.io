## Overview

This is a personal academic website for Alec Plotkin, built on the [Academic Pages](https://academicpages.github.io/) Jekyll template (forked from Minimal Mistakes). It is hosted on GitHub Pages at `https://alecplotkin.github.io`.

## Development Commands

**Serve locally with live reload:**
```bash
bundle exec jekyll serve -l -H localhost
```
The site is available at `http://localhost:4000`. Note: changes to `_config.yml` require restarting the server; Markdown and HTML changes auto-reload.

**Install dependencies:**
```bash
bundle install
```
If you encounter permission errors, run `bundle config set --local path 'vendor/bundle'` first.

**Using Docker instead:**
```bash
chmod -R 777 .
docker compose up
```

**Update CV JSON from markdown:**
```bash
bash scripts/update_cv_json.sh
```
This converts `_pages/cv.md` → `_data/cv.json` via `scripts/cv_markdown_to_json.py`.

## Architecture

### Content Collections

Content lives in Jekyll collections — directories prefixed with `_`. Each collection item is a Markdown file with YAML front matter:

- `_publications/` — Academic papers (fields: `title`, `date`, `venue`, `paperurl`, `citation`, `excerpt`, `category`, `collection`)
- `_talks/` — Conference talks (fields: `title`, `date`, `venue`, `location`, `type`)
- `_teaching/` — Teaching experience
- `_portfolio/` — Portfolio items
- `_pages/` — Static pages (about, cv, publications listing, etc.)

### Key Configuration Files

- `_config.yml` — Site-wide settings: author info, social links, collection definitions, Jekyll plugins. Author bio/avatar/social links are all set here.
- `_data/navigation.yml` — Controls which pages appear in the header nav. Items are commented out to hide them without deleting.
- `_data/cv.json` — Machine-readable CV (JSON Resume format), auto-generated from `_pages/cv.md` via scripts.

### Page Listings

Collection index pages in `_pages/` (e.g., `publications.html`, `talks.html`) use Liquid templates to iterate over their respective collections. The `_pages/about.md` is the homepage (`permalink: /`).

### CV Dual Format

There are two CV options available:
- `_pages/cv.md` — Markdown-formatted CV (manually maintained)
- `_pages/cv-json.md` — CV rendered from `_data/cv.json` (generated by running `scripts/update_cv_json.sh`)

Only one should be linked in `_data/navigation.yml` at a time.

### Markdown Generators

`markdown_generator/` contains Python scripts and Jupyter notebooks to bulk-generate collection Markdown files from TSV/CSV data:
- `publications.py` / `publications.ipynb` — from `publications.tsv` or `publications.csv`
- `talks.py` / `talks.ipynb` — from `talks.tsv`
- `PubsFromBib.ipynb` / `pubsFromBib.py` — from BibTeX
- `OrcidToBib.ipynb` — from ORCID

### CI/CD

`.github/workflows/scrape_talks.yml` — Triggered on pushes that modify `_talks/` or `talkmap.ipynb`. Runs the `talkmap.ipynb` notebook to geocode talk locations and commits the output (`talkmap_out.ipynb`) and updated `talkmap/org-locations.js` back to the repo.

### Deployment

Pushing to `master` triggers GitHub Pages to build and deploy the site automatically. The `_site/` directory is the local build output (not committed).
